En esta carpeta o capa, debemos definir los middlewares que vamos a usar en el proyecto.
1.- Implementar un middleware de autenticacion
2.- JWT, Bearer Token y las validaciones -> ADASDASDASD -> Informacion del usuario
3.- Implementar un middleware de autorizacion
3.1.- Es resolver en un solo middleware todas las condiciones
3.1.1.- Que el servicio este disponible solo para admin
3.1.2.- Que el usuario sea due単o de la cuenta
3.1.3.- Que el usuario sea due単o del departamento -> Van a necesitar consultar informacion a la BDD

- Primero necesito saber cual es el usuario que esta haciendo el request -> lo obtengo del middleware de autenticacion
- Luego comparar esos ids, en el caso que haga match le dejamos continuar o en el caso que no un 403

- NOTA: Si necesitan hacer alguna consulta a BDD, en el middleware pueden llamar al modelo o modelos que necesiten

- 2 opciones para resolver este flujo:

1.- Lo implementan todo dentro del mismo middleware de autorizacion
2.- Lo implentan en middlewares separados, middleware para validacion de admin, due単o de la cuenta, usuario due単o del departamento
